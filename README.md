# DevOps DB Project

**Курсовой проект:** Интеграция Open-Source БД с инструментами DevOps и конвейерами CI/CD.

**Проект летней практики:** Разработка веб-интерфейса управления пользователями с ролевой моделью доступа, системой резервного копирования и подсистемой аудита.

Данный репозиторий содержит микросервис управления пользователями, разработанный для демонстрации концепции Database DevOps и построения внутренних панелей администрирования. Проект включает в себя настроенный полный цикл CI/CD, управление схемой БД, веб-интерфейс с разделением прав, систему бэкапов, интеграционное тестирование и стек мониторинга с настроенными оповещениями.

## Ключевые возможности

* **Ролевая модель:** Панель управления с разделением прав доступа (`admin` и `manager`).
* **Управление данными:** Импорт пользователей (CSV/XLSX), обработка запросов на удаление.
* **Безопасность и аудит:** Шифрование персональных данных (ФИО) на уровне приложения, ведение подробного журнала аудита всех ручных действий и изменений.
* **Резервное копирование:** Встроенный механизм автоматического и ручного создания/восстановления бэкапов БД.
* **Мониторинг и алертинг:** Сбор метрик приложения и БД через Prometheus, визуализация дашбордов в Grafana с настроенными правилами оповещения (alerts) об инцидентах.
* **Непрерывная интеграция и доставка (CI/CD):** Автоматизированное тестирование, линтинг и деплой проекта.

## Стек технологий

* **Backend & Admin:** Python 3.12, FastAPI, SQLAlchemy 2.0, SQLAdmin
* **База данных:** PostgreSQL 16, Alembic (управление миграциями) 
* **Тестирование & Code Quality:** Pytest, Testcontainers, Линтеры (Flake8/Black)
* **Инфраструктура:** Docker, Docker Compose 
* **CI/CD:** GitHub Actions 
* **Мониторинг:** Prometheus, Grafana (Dashboards & Alerting), `postgres_exporter`

## Быстрый старт

1. Склонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/Lifor121/devops-db-project.git
cd devops-db-project
```

2. Запустите инфраструктуру (веб-сервис, БД и стек мониторинга) в фоне:

```bash
docker compose up --build -d
```

**Доступные эндпоинты после запуска:**

* Панель администрирования: `http://localhost:8000/admin` *(логины: admin/admin или manager/manager)*
* Приложение (Health Check): `http://localhost:8000/health` 
* Интерактивная документация API (Swagger): `http://localhost:8000/docs` 
* Prometheus: `http://localhost:9090` 
* Grafana: `http://localhost:3000` *(логин/пароль по умолчанию: admin/admin)* 

## Работа с миграциями (Alembic)

Проект использует декларативный подход к моделям и автогенерацию миграций.

* **Генерация новой миграции** (после изменения моделей в `models.py`):

```bash
alembic revision --autogenerate -m "описание_изменений"

```

* **Применение миграций к БД**:

```bash
alembic upgrade head
```

* **Откат последней миграции**:

```bash
alembic downgrade -1
```

## Тестирование

Для безопасного интеграционного тестирования без засорения основной базы используется `testcontainers`. При запуске тестов автоматически поднимается и затем уничтожается изолированный Docker-контейнер с PostgreSQL.

Для запуска автотестов выполните:

```bash
pytest -v
```

## CI/CD Конвейер

Пайплайн настроен в `.github/workflows/main.yml`.
При каждом коммите или Pull Request в ветку `master` GitHub Actions автоматически выполняет:

1. Развертывание тестовой среды.
2. Проверка кода линтерами на соответствие стандартам.
3. Проверку конфигурации и применение миграций Alembic.
4. Запуск интеграционных тестов.
5. При успешном прохождении всех проверок запускается автоматическое развертывание обновленной версии приложения на сервере.
